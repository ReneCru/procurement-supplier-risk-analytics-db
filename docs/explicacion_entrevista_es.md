# Explicación para Entrevista

## Explicación Corta

Construí una base de datos relacional en PostgreSQL para analizar procurement y riesgo de proveedores.

El modelo incluye proveedores, órdenes de compra, líneas de orden de compra, entregas, facturas, compradores, categorías y documentos de compliance.

El proyecto usa datos públicos de procurement como base y genera datos operativos sintéticos para simular entregas, facturas y documentación de compliance.

## Valor de Negocio

La base de datos permite analizar:

- proveedores con mayor gasto;
- proveedores con mayor riesgo de entregas tardías;
- órdenes de compra abiertas por antigüedad;
- documentos de compliance vencidos o faltantes;
- gasto por categoría;
- tendencia mensual de compras;
- priorización de proveedores por nivel de riesgo.

## Por Qué Este Proyecto Es Relevante

Este proyecto conecta SQL y diseño de bases de datos con problemas reales de compras, abastecimiento, supply chain y compliance.

Demuestra capacidad para:

- diseñar una base de datos relacional;
- crear llaves primarias y foráneas;
- aplicar restricciones de calidad de datos;
- crear índices para análisis;
- transformar datos públicos en un modelo de negocio;
- construir KPIs de procurement;
- explicar trabajo técnico con enfoque ejecutivo.

## Uso Ético de Datos

Este proyecto no utiliza información confidencial de ninguna empresa.

Los datos públicos se usan para proveedores, compradores, fechas, categorías y montos. Los datos operativos como entregas, facturas y documentos de compliance son sintéticos y fueron generados con reglas de negocio documentadas.
